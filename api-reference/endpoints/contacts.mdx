---
title: 'Contacts'
description: 'Create, retrieve, update, and delete contacts'
---

Contacts represent the people in your project. Every email address you send to or track an event for is automatically upserted as a contact.

<Note>
  All contact endpoints require a **secret key** (`sk_*`).
</Note>

---

## List contacts

`GET /contacts`

Returns a cursor-paginated list of contacts for the authenticated project.

### Query parameters

<ParamField query="limit" type="number" default="20">
  Number of contacts to return per page. Maximum `100`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor returned from a previous response. Omit on the first request.
</ParamField>

<ParamField query="search" type="string">
  Filter contacts by email address (partial match).
</ParamField>

### Response

<ResponseField name="contacts" type="array" required>
  Array of contact objects.

  <Expandable title="Contact properties">
    <ResponseField name="id" type="string">Contact ID.</ResponseField>
    <ResponseField name="email" type="string">Contact email address.</ResponseField>
    <ResponseField name="subscribed" type="boolean">Whether the contact is subscribed to marketing emails.</ResponseField>
    <ResponseField name="data" type="object">Custom data fields stored on the contact.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 creation timestamp.</ResponseField>
    <ResponseField name="updatedAt" type="string">ISO 8601 last-updated timestamp.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="cursor" type="string">
  Cursor to pass as `cursor` on the next request. `null` when there are no more pages.
</ResponseField>

<ResponseField name="hasMore" type="boolean">
  `true` when additional pages are available.
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of contacts. Only included on the first page (when no `cursor` is provided).
</ResponseField>

```bash Example
curl --request GET \
  --url 'https://next-api.useplunk.com/contacts?limit=20' \
  --header 'Authorization: Bearer sk_live_yourkey'
```

```json 200
{
  "contacts": [
    {
      "id": "clx_contact_abc123",
      "email": "user@example.com",
      "subscribed": true,
      "data": { "plan": "pro" },
      "createdAt": "2024-01-01T00:00:00.000Z",
      "updatedAt": "2024-01-10T08:00:00.000Z"
    }
  ],
  "cursor": "clx_contact_def456",
  "hasMore": true,
  "total": 1042
}
```

## Create contact

`POST /contacts`

Creates a new contact or updates an existing one (upsert by email address).

### Body parameters

<ParamField body="email" type="string" required>
  Contact email address. Used as the unique identifier for upsert.
</ParamField>

<ParamField body="subscribed" type="boolean" default="true">
  Subscription status for marketing emails.
</ParamField>

<ParamField body="data" type="object">
  Custom key-value data to store on the contact. Merged with any existing data.
</ParamField>

### Response

Returns the contact object. HTTP `201` when a new contact was created; `200` when an existing contact was updated.

<ResponseField name="id" type="string">Contact ID.</ResponseField>
<ResponseField name="email" type="string">Contact email address.</ResponseField>
<ResponseField name="subscribed" type="boolean">Subscription status.</ResponseField>
<ResponseField name="data" type="object">Custom contact data.</ResponseField>
<ResponseField name="createdAt" type="string">ISO 8601 creation timestamp.</ResponseField>
<ResponseField name="updatedAt" type="string">ISO 8601 last-updated timestamp.</ResponseField>
<ResponseField name="_meta" type="object">
  <Expandable title="properties">
    <ResponseField name="isNew" type="boolean">
      `true` when the contact was just created.
    </ResponseField>
    <ResponseField name="isUpdate" type="boolean">
      `true` when an existing contact was updated.
    </ResponseField>
  </Expandable>
</ResponseField>

```bash Example
curl --request POST \
  --url https://next-api.useplunk.com/contacts \
  --header 'Authorization: Bearer sk_live_yourkey' \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "user@example.com",
    "subscribed": true,
    "data": {
      "firstName": "Jane",
      "plan": "premium"
    }
  }'
```

```json 201
{
  "id": "clx_contact_abc123",
  "email": "user@example.com",
  "subscribed": true,
  "data": { "firstName": "Jane", "plan": "premium" },
  "createdAt": "2024-01-15T10:30:00.000Z",
  "updatedAt": "2024-01-15T10:30:00.000Z",
  "_meta": { "isNew": true, "isUpdate": false }
}
```

## Update contact

`PATCH /contacts/:id`

Updates a contact's subscription status or custom data fields.

### Path parameters

<ParamField path="id" type="string" required>
  The contact ID.
</ParamField>

### Body parameters

<ParamField body="email" type="string">
  New email address for the contact.
</ParamField>

<ParamField body="subscribed" type="boolean">
  Updated subscription status.
</ParamField>

<ParamField body="data" type="object">
  Custom data fields to merge into the contact's existing data.
</ParamField>

### Response

Returns the updated contact object.

```bash Example
curl --request PATCH \
  --url https://next-api.useplunk.com/contacts/clx_contact_abc123 \
  --header 'Authorization: Bearer sk_live_yourkey' \
  --header 'Content-Type: application/json' \
  --data '{
    "subscribed": false,
    "data": { "plan": "free" }
  }'
```

```json 200
{
  "id": "clx_contact_abc123",
  "email": "user@example.com",
  "subscribed": false,
  "data": { "plan": "free" },
  "createdAt": "2024-01-01T00:00:00.000Z",
  "updatedAt": "2024-01-15T12:00:00.000Z"
}
```

## Delete contact

`DELETE /contacts/:id`

Permanently deletes a contact and all associated data.

<Warning>
  This action is irreversible. All events, email history, and segment memberships for the contact are removed.
</Warning>

### Path parameters

<ParamField path="id" type="string" required>
  The contact ID.
</ParamField>

### Response

Returns `204 No Content` on success.

```bash Example
curl --request DELETE \
  --url https://next-api.useplunk.com/contacts/clx_contact_abc123 \
  --header 'Authorization: Bearer sk_live_yourkey'
```

## Import contacts (CSV)

`POST /contacts/import`

Queues a CSV file for bulk contact import. The import runs asynchronously in the background.

<Note>
  The request must use `multipart/form-data` with the CSV file in a field named `file`. Maximum file size is 5 MB. The CSV must have at minimum an `email` column.
</Note>

### Body (multipart/form-data)

<ParamField body="file" type="file" required>
  CSV file. Must include an `email` column. Additional columns are stored as contact data fields.
</ParamField>

### Response

Returns `202 Accepted` with a `jobId` you can poll for status.

<ResponseField name="message" type="string">Confirmation message.</ResponseField>
<ResponseField name="jobId" type="string">Use this to check import progress.</ResponseField>

```bash Example
curl --request POST \
  --url https://next-api.useplunk.com/contacts/import \
  --header 'Authorization: Bearer sk_live_yourkey' \
  --form 'file=@contacts.csv'
```

## Bulk subscribe

`POST /contacts/bulk-subscribe`

Queues a bulk subscribe operation for up to 1,000 contacts.

### Body parameters

<ParamField body="contactIds" type="string[]" required>
  Array of contact IDs to subscribe. Maximum 1,000 per request.
</ParamField>

### Response

Returns `202 Accepted` with a `jobId`.

```bash Example
curl --request POST \
  --url https://next-api.useplunk.com/contacts/bulk-subscribe \
  --header 'Authorization: Bearer sk_live_yourkey' \
  --header 'Content-Type: application/json' \
  --data '{ "contactIds": ["clx_abc", "clx_def"] }'
```
